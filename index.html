Overview

How a detection moves through the lab

  1. Endpoints generate normal + malicious traffic from the isolated VMs.
  2. Forwarder ships logs to the central Splunk indexer in real time.
  3. Suricata flags scans and known-bad patterns against signature + Sigma rules.
  4. Shuffle picks up the alert and runs an automated triage playbook.
  5. VirusTotal enriches the alert with global threat intelligence.
  6. Splunk surfaces the correlated, enriched alert on the analyst dashboard.

THE TOOLKIT

What's actually running in the lab

Every tool here is wired into the pipeline — nothing is a standalone demo. Logs, alerts, and enrichment all flow into one dashboard.

Splunk SIEM

Detection & Triage

Where everything lands — searchable logs, correlation rules, and dashboards an analyst would actually work from.

Suricata IDS

Network Detection

Inspects traffic against signature rules and custom Sigma rules to flag scans, exploits, and known-bad patterns.

Nmap

Recon Simulation

Run from Kali to simulate reconnaissance — its scan signatures are exactly what Suricata is tuned to catch.

VirusTotal

Threat Enrichment

Shuffle calls the VirusTotal API automatically to check file hashes and IPs against global threat intelligence.

Shuffle SOAR

Automated Response

Picks up Suricata alerts and runs a full playbook — enrich, notify, and block — with no human in the loop.

Hydra

Brute-Force Simulation

Simulates credential-stuffing and brute-force login attempts to validate alerting thresholds end to end.
